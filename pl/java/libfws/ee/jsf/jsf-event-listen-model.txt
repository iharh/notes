    three types of events: 
        application events (obsolete)
            re tied to a particular application and are generated by a UIComponent
            represent the standard events available in prev versions of JSF
        system events
            are generated by an Object rather than a UIComponent. 
            are generated during the execution of an application at predefined times. 
            are applicable to the entire application rather than to a specific component.
        data-model events
            occurs when a new row of a UIData component is selected.

    2 kinds of events:
        action event occurs when the user activates a component that implements ActionSource. 
        These components include buttons and links.
        class: jakarta.faces.event.ActionEvent

        A value-change event (class jakarta.faces.event.ValueChangeEvent) 
        occurs when the user changes the value of a component represented by UIInput or one of its subclasses

    the "immediate" attribute
    https://jakarta.ee/learn/docs/jakartaee-tutorial/current/web/faces-page/faces-page.html#_the_immediate_attribute
        Depending on the value of the immediate property (see The immediate Attribute) of the component emitting the event, 
        action events can be processed during the Invoke Application phase or the Apply Request Values phase, 
        and value-change events can be processed during the Process Validations phase or the Apply Request Values phase.


There are two ways to cause your application to react to action events or value-change events that are emitted by a standard component:
    * Implement an event listener class to handle the event, 
    https://jakarta.ee/learn/docs/jakartaee-tutorial/current/web/faces-custom/faces-custom.html#_implementing_an_event_listener
    and register the listener on the component by nesting either an 
        f:valueChangeListener tag or an f:actionListener tag 
    inside the component tag.

    * Implement a method of a managed bean to handle the event, 
    https://jakarta.ee/learn/docs/jakartaee-tutorial/current/web/faces-page-core/faces-page-core.html#_registering_listeners_on_components
    and refer to the method with a method expression from the appropriate attribute of the component’s tag.

https://jakarta.ee/learn/docs/jakartaee-tutorial/current/web/faces-page-core/faces-page-core.html#_registering_listeners_on_components

    <f:setPropertyActionListener target="#{requestScope.book}" value="#{book}"/>
